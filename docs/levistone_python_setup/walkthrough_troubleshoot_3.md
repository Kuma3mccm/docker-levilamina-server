# LeviStoneフリーズ問題のトラブルシューティング(3)

前回の修正でパイプ入力を排除したところ、今度はサーバープロセスが「標準入力が閉じられた」と判定して終了(`Exit code: 3`)してしまいました。
これを踏まえ、サーバーが即座に終了するのを防ぎつつ、コンソールからの入力（`cat`）によるフリーズを回避する方法を試みます。

## 実施した対策

`wine/entrypoint.sh` の最下部の起動コマンドを以下のように変更しました。

```diff
- wine bedrock_server_mod.exe
+ (tail -f /dev/null | wine bedrock_server_mod.exe) 2>&1
```

これにより、標準入力ストリームは開かれたままになりますが、実際のコンソールからの入力はすべて無視される（空データが入力され続ける）状態になります。
この状態でサーバーがフリーズせずに起動を完了（`Server started.` などのログが表示）できるかを検証します。

**注意点**:
この修正により、Pterodactylパネルのコンソールから「stop」などのコマンドを送信してもサーバー側は受け付けなくなります。
今回サーバーが正常に起動した場合は、コンソール入力をどのようにしてサーバーへ送り込むか（`wineconsole` の使用など）の追加対策を検討します。
