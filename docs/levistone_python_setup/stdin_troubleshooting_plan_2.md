# LeviStoneフリーズ問題の追加トラブルシューティング (3)

`cat |` パイプを外したことでサーバーが即落ち (`Exit code: 3`) してしまうことが判明したため、Bedrockサーバーは標準入力が閉じられるとエラーで終了してしまう仕様であることが分かりました。

しかし、`cat |` で標準入力を開きっぱなしにするとLevistone(Python)側の起動フェーズで何らかのデッドロックが生じている（フリーズする）状態です。

## 修正計画

この問題が「`cat`からのパイプ入力」と「Pythonプロセス」間のTTYハンドリングの問題なのかを切り分けるため、`cat` の代わりに `tail -f /dev/null` を使ってダミーの入力を流し続ける手法を試します。
これにより、サーバープロセスへの標準入力ストリームは開かれたままになりますが、コンソールからの入力は受け付けなくなります。
もしこれでサーバーが正常に起動完了（"Server started." 等が表示）した場合、原因は「パイプ経由のコンソール入力機構とLevistoneとの相性」に絞られます。

```sh
# entrypoint.sh 変更
- wine bedrock_server_mod.exe
+ (tail -f /dev/null | wine bedrock_server_mod.exe) 2>&1
```

もしこれでもフリーズするようであれば、Levistone側の問題（依存関係やWine特有の不具合）である可能性がさらに高まります。
