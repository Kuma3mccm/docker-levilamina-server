# LeviStoneフリーズ問題のトラブルシューティング(2)

LeviStoneがロードされた直後にサーバーがフリーズ（ハング）する問題に対して、標準入力（TTY）の競合によるデッドロックが疑われました。

## 実施した対策

`wine/entrypoint.sh` の最下部にある `bedrock_server_mod.exe` の起動部分において、パイプを使って標準入力ストリームを割り当てていた `(cat | ...)` と `2>&1` の記述を削除し、直接 `wine bedrock_server_mod.exe` を実行するように変更しました。

Python（LeviStone）がWine上でロードされた際、コンソール入力を待ち受けてしまったり、`cat`プロセスにブロックされてしまう問題を回避できるか検証するための措置です。

## 確認のお願い

上記の修正が適用された状態でもう一度コンテナの再起動（サーバーの起動）をお試しください。
もしこれで無事にサーバーが立ち上がった場合は、Pterodactylコンソール上で「stop」などのコマンドが打てるかどうかも合わせてご確認ください。
