# LeviStoneフリーズ問題のトラブルシューティング

LeviStoneがロードされた直後にサーバーがフリーズ（ハング）する問題に対して、`wine/entrypoint.sh` にいくつかの回避策とデバッグ設定を適用しました。

## 実施した対策

1. **Wineクラッシュダイアログの無効化**
   Wine環境下でPythonが例外を投げたりクラッシュしたりした際、エラーダイアログが出現しようとしてターミナル（CUI）上で処理がブロックされてしまうことがあります。
   これを防ぐため、`ShowCrashDialog` をレジストリで無効化し、エラーがログとしてコンソールに落ちるようにしました。
2. **Pythonモジュールパスの強制解決**
   Python Embeddableパッケージの挙動（`._pth`ファイルが存在する場合、`PYTHONPATH` などの環境変数が無視される仕様）により、`plugins/EndstoneRuntime` フォルダにインストールされたモジュール（Levistone本体など）が見つからないエラーが起きていた可能性があります。
   そのため、起動時に `python311._pth` ファイルの末尾に `Z:\home\container\plugins\EndstoneRuntime` を直接追記する処理を追加しました。これにより、LeviLaminaからホストされるPythonランタイムが必要なモジュールをインポートできるようになります。
3. **入出力バッファリングの無効化**
   標準入出力がブロックされないよう、`PYTHONUNBUFFERED=1` 環境変数を追加しました。

## 確認のお願い

上記の修正が適用された状態でもう一度起動（コンテナ再起動）をお試しください。
問題が解決してサーバーが立ち上がるか、もしくは「ImportError」や「Fatal Python error」などの明確なエラーメッセージがログに表示されるようになる可能性があります。
稼働状況、または新しいエラーログが出た場合はお知らせください。
